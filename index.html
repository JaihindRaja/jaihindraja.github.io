Methodology

System Thinking

Over the past few years working in logistics technology, I've learned that most production failures are not simple bugs — they are system behavior problems. My approach focuses on tracing problems across the full platform rather than investigating isolated components.

STEP · 01
Signal Detection
Identify anomalies through production alerts, driver reports, support tickets, or operational inconsistencies. The goal is to detect patterns rather than isolated bugs.
STEP · 02
System Trace
Trace the full system path across APIs, databases, services, and external integrations to understand where the breakdown occurs.
STEP · 03
Root Cause Analysis
Analyze logs, payloads, and system behavior to identify whether the issue originates from code logic, data inconsistencies, external integrations, or operational workflows.
STEP · 04
Solution Design
Propose improvements such as payload optimization, system safeguards, better monitoring, or workflow changes to prevent similar failures.

Driver App Crash Investigation
Problem
Driver app was crashing during bulk pickup assignment events in production, affecting delivery operations.
Investigation
Traced crash timing against bulk assignment events. Found the order assignment logic was loading an oversized data payload in a single request.
Solution
Recommended pagination and progressive loading for order assignment responses to reduce payload size per request.
Impact
Identified root cause, reduced crash frequency, and improved driver app stability during high-assignment periods.
Zone Routing Failure Analysis
Problem
Zone routing feature was underperforming — deliveries weren't following zone assignments despite the feature being technically active.
Investigation
Analysis revealed the failure was behavioral, not technical — driver adoption friction and operational complexity caused zone bypass.
Solution
Explored OSRM-based routing using actual road network data to redefine zone boundaries, reducing the gap between planned and actual routes.
Impact
Documented systemic gap between technical design and operational reality; proposed data-driven zone improvement approach.
Production Debugging & Integration Issues
Problem
Recurring intermittent production failures involving third-party integrations and webhook delivery — hard to reproduce.
Investigation
Systematic log analysis to identify timing patterns, payload mismatches, and failure states. Distinguished platform failures from external service issues.
Solution
Collaborated with engineers to narrow root causes; documented findings for incident response and improved monitoring coverage.
Impact
Improved platform reliability and reduced mean resolution time for similar future incidents.
Address Book Integration Optimization
Problem
Drivers were manually entering addresses, causing incorrect locations and delivery failures that impacted last-mile success rates.
Solution
Tested and validated Address Book integration with WhatsApp and Ola data sources to auto-populate customer addresses, eliminating manual entry errors.
Impact
Improved address accuracy and increased delivery success rate to approximately 85% in last-mile logistics.
Contribution
Provided product feedback after understanding the system end-to-end and identifying edge cases in address validation logic.
